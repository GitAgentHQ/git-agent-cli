AI-first Git CLI for automated commit message generation using LLMs.

## Installation

```bash
go install github.com/fradser/ga-cli@latest
```

Or download a pre-built binary from the [releases](https://github.com/fradser/ga-cli/releases).

## Quick Start

```bash
# Initialize ga in your repository
ga init

# Stage your changes
ga add .

# Generate and create a commit with AI-generated message
ga commit

# Or stage all changes and commit in one step
ga commit --all
```

## Commands

### `ga init`

Initialize ga in the current repository. Analyzes git history and directory structure to generate project-specific commit scopes.

```bash
ga init                    # Use default empty hook
ga init --hook conventional  # Install conventional commit validator
ga init --force            # Overwrite existing config
```

### `ga commit`

Generate and create a commit with an AI-generated message.

```bash
ga commit                  # Commit with staged changes
ga commit --all           # Stage all changes first, then commit
ga commit --dry-run       # Print message without committing
ga commit --intent "fix auth bug"  # Provide context hint to the LLM
```

### `ga add`

Stage files for commit (wrapper around `git add`).

```bash
ga add .
ga add src/ utils/
```

## Configuration

### User Config (`~/.config/ga/config.yml`)

Optional. Point to any OpenAI-compatible endpoint:

```yaml
base_url: https://api.openai.com/v1
api_key: sk-...
model: gpt-4o
```

Examples for other providers:

```yaml
# Cloudflare Workers AI
base_url: https://api.cloudflare.com/client/v4/accounts/YOUR_ACCOUNT_ID/ai/v1
api_key: YOUR_CLOUDFLARE_API_TOKEN
model: "@cf/meta/llama-3.1-8b-instruct"
```

```yaml
# Local Ollama
base_url: http://localhost:11434/v1
model: llama3
```

### Project Config (`.ga/project.yml`)

Generated by `ga init`. Defines team-scoped commit types:

```yaml
scopes:
  - api
  - core
  - auth
  - infra
```

### Hooks

Built-in hooks (installed by `ga init --hook <name>`):

| Hook | Description |
|------|-------------|
| `empty` | Default placeholder, always passes |
| `conventional` | Validates Conventional Commits format |

Custom hooks are executable scripts at `.ga/hooks/pre-commit`. They receive JSON via stdin and should exit 0 to proceed or non-zero to block.

## Flags

| Flag | Description |
|------|-------------|
| `-v, --verbose` | Enable verbose output |
| `--api-key` | API key for the AI provider |
| `--model` | Model to use for generation |
| `--base-url` | Base URL for the AI provider |
| `--max-diff-lines` | Maximum diff lines to send to the model (default: 500) |
| `--all, -a` | Stage all tracked changes before committing |
| `--dry-run` | Print commit message without committing |
| `--intent` | Describe the intent of the change |
| `--co-author` | Add a co-author to the commit message |

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success — commit created (or dry-run complete) |
| 1 | General error — no staged changes, API failure, missing config |
| 2 | Hook blocked — pre-commit hook returned non-zero |
